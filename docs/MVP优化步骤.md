# Cursor Skin Manager MVP 优化步骤

本文档基于当前 MVP 定位整理：只做本地 Windows 光标皮肤管理，不扩展在线市场、账号、收藏、标签等功能。

## MVP 目标

应用需要稳定完成以下闭环：

1. 导入本地皮肤包。
2. 自动识别 `.cur` / `.ani` / `.inf`。
3. 预览整套光标。
4. 一键应用到 Windows 当前用户。
5. 删除皮肤包。
6. 必要时恢复 Windows 默认鼠标光标。

## 已确认决策

1. 删除已应用皮肤：自动先恢复默认，再删除。
2. “恢复默认”全局入口：放在左侧侧边栏。
3. 判断皮肤是否已应用：全部已设置角色都匹配才算已应用。

## 第一阶段：应用与删除安全性

目标：避免用户把系统光标指向已经不存在的文件。

执行步骤：

1. 后端删除皮肤前读取最新皮肤库状态。
2. 根据 Windows 当前注册表同步 `isApplied`。
3. 如果待删除皮肤正在应用，先恢复 Windows 默认光标。
4. 默认光标恢复成功后，再删除皮肤目录。
5. 删除完成后更新 `library.json`。
6. 如果恢复默认失败，停止删除并返回错误。

验收标准：

- 删除未应用皮肤：直接删除。
- 删除已应用皮肤：系统先恢复默认，然后皮肤从列表消失。
- 恢复默认失败时：皮肤不删除，界面显示错误。

## 第二阶段：恢复默认做成全局功能

目标：即使没有选中皮肤，用户也能恢复 Windows 默认光标。

执行步骤：

1. 将“恢复默认”入口移动到左侧侧边栏。
2. 保证皮肤列表为空时也可以点击。
3. 恢复默认前显示二次确认。
4. 恢复默认成功后，所有皮肤状态改成“未应用”。

验收标准：

- 有选中皮肤时，可以从左侧恢复默认。
- 无皮肤或空状态时，也可以从左侧恢复默认。
- 恢复后右侧状态卡不再显示任何皮肤已应用。

## 第三阶段：真实检测当前系统应用状态

目标：应用显示的“已应用/未应用”应和 Windows 当前用户注册表一致。

执行步骤：

1. `load_library` 时读取 `HKCU\Control Panel\Cursors`。
2. 遍历每个皮肤包的已设置角色。
3. 全部已设置角色都匹配当前注册表路径时，判定该皮肤已应用。
4. 没有任何皮肤完全匹配时，全部标记为未应用。
5. 写入 `library.json` 前清理预览 data URL，避免文件膨胀。

验收标准：

- 用户在 Windows 鼠标设置中改掉光标后，重新打开应用不再错误显示旧皮肤已应用。
- 应用某套皮肤后，重新打开应用仍能识别它已应用。
- 恢复默认后，重新打开应用不会显示任何本地皮肤正在使用。

## 第四阶段：导入体验优化（已完成基础版）

目标：减少错误导入和重复导入。

已完成：

1. 导入 `.inf` 时，不再直接复制整个父目录。
2. 优先解析 `.inf` 引用到的 `.cur` / `.ani`。
3. 只复制必要文件和 `.inf`。
4. 导入前检测是否已经导入过同一路径。

后续可继续：

1. 如果 `.inf` 解析失败，提示用户改用“导入文件夹”。
2. 增加同名同文件数量的重复皮肤检测。

## 第五阶段：预览兼容性增强（已完成基础版）

目标：提高 `.cur` / `.ani` 预览成功率。

已完成：

1. 支持 1 位、4 位、8 位索引色 DIB。
2. 正确处理调色板。
3. 正确处理 AND mask 透明区域。
4. `.ani` 多帧时优先取首个有效 icon chunk。
5. 预览失败时显示更自然的占位符。

验证：

- 已增加 1/4/8 位索引色 DIB 单元测试。
- 三种格式均能生成正确尺寸和颜色的 PNG。
- 现有 PNG、24 位、32 位预览逻辑保持不变。

## 第六阶段：数据可靠性（已完成）

目标：避免 `library.json` 损坏导致整个应用无法读取。

已完成：

1. 写入 `library.json` 前先写临时文件。
2. 写入成功后再替换原文件。
3. 替换前保留 `library.bak.json`。
4. 读取失败时尝试恢复备份。
5. 如果备份也失败，则重建空库，并保留损坏文件用于排查。

恢复文件说明：

- 主数据：`library.json`
- 最近一次备份：`library.bak.json`
- 损坏主文件：`library.corrupt-*.json`
- 损坏备份：`library.backup-corrupt-*.json`

验证：

- 已增加安全写入与备份单元测试。
- Rust 单元测试和编译检查通过。

## 第七阶段：交互收敛（已完成）

目标：MVP 界面更清楚，不增加复杂功能。

已完成：

1. “刷新光标”改名为“重新加载光标”。
2. 应用成功后按钮短暂显示“已应用”。
3. 恢复默认成功后按钮短暂显示“已恢复”。
4. 不完整皮肤包应用前显示会应用几个角色。

交互说明：

- 成功反馈显示约 1.6 秒，不额外增加右下角提示框。
- 连续操作会清理上一次反馈计时器，避免状态互相覆盖。
- 不完整皮肤会明确显示 `已匹配数量 / 15`，未设置角色保持系统当前配置。

## 第八阶段：应用设置与托盘（已完成）

已确认并完成：

1. 开机自启，默认关闭。
2. 设置页可打开应用数据目录，查看皮肤文件、数据库、备份和日志。
3. 设置页可清空全部本地皮肤，操作前进行两次强确认。
4. “清空全部皮肤”按钮旁显示风险提示；如果当前皮肤正在使用，会先恢复 Windows 默认光标。
5. 关闭窗口时最小化到系统托盘，默认开启。
6. 托盘左键可重新显示主窗口，托盘菜单提供“显示主窗口”和“退出”。
7. 设置入口使用齿轮图标并固定在左侧栏底部。
8. “恢复默认光标”移动到设置页的“数据与维护”区域。
9. 详情页底部工具栏已移除。
10. “打开皮肤目录”移动到详情页顶部，紧邻“重新加载光标”，只打开当前选中皮肤的内部目录。
11. “打开日志”移动到设置页的“数据与维护”区域。

设置存储：

- 设置文件：`settings.json`
- 设置备份：`settings.bak.json`
- 设置文件损坏时自动恢复默认配置。

默认值：

- `开机自启 = 关闭`
- `关闭窗口时最小化到托盘 = 开启`

## 推荐执行顺序

1. 删除已应用皮肤的安全处理。
2. 恢复默认入口全局化。
3. 启动时真实检测当前系统光标状态。
4. 导入 `.inf` 的复制策略优化。
5. 预览解析增强。
6. `library.json` 备份恢复。
7. 交互文案微调。

## 跨电脑启动兼容性（已完成基础修复）

已完成：

1. Windows 安装包使用 WebView2 `offlineInstaller`，目标电脑无需联网下载运行库。
2. 托盘初始化失败时降级为普通窗口模式，不再终止应用。
3. 新增独立启动日志 `%AppData%\CursorSkinManager\startup.log`。
4. 捕获启动期和运行期 panic，并显示原生 Windows 错误框。
5. Tauri 运行时启动失败时显示错误原因和诊断日志路径。

发布说明：

- 离线兼容安装包约 199 MB。
- 便携版仍要求目标电脑已经安装 Microsoft Edge WebView2 Runtime。
- 当前安装包未进行数字签名，正式公开分发前仍需准备 Windows 代码签名证书。

## 应用图标清晰度（已完成）

1. 小尺寸图标改为扁平高对比设计，移除阴影和细碎装饰。
2. ICO 包含 16、20、24、32、40、48、64、128、256px 九档图层。
3. 16–24px 图层使用像素对齐坐标，改善任务栏和系统托盘清晰度。
4. 保留 `icon-master.png` 和 `scripts/generate-icon.ps1`，后续可以重复生成全部尺寸图标。
5. 应用版本升级到 `0.1.1`，减少 Windows 沿用旧图标缓存的情况。

## 光标角色替换与重新分配（0.1.10，已完成）

1. 整套预览中的 15 个角色卡片支持 hover，卡片内操作按钮支持键盘 focus，可通过原生文件选择器替换 `.cur` / `.ani`。
2. Rust 后端在复制前验证文件存在性、扩展名、CUR/ANI 结构、文件大小和真实预览。
3. 外部文件复制到当前皮肤的 `custom` 目录，同名文件使用递增名称且不覆盖已有文件。
4. 被替换文件没有被其他角色引用时进入未分配列表；共享文件继续保留原角色映射且不重复添加。
5. 未分配文件可通过正式角色选择弹窗分配到任意角色，已有角色文件按相同规则交换回未分配列表。
6. 弹窗支持 `Esc`、`Tab` 焦点循环、`Enter` 选择与确认，操作期间禁用重复提交。
7. 新增 `replace_cursor_role` 和 `assign_unassigned_cursor` Tauri command，所有映射更新均由 Rust 完成。
8. 文件复制、预览生成和 `library.json` 原子写入组成事务；保存失败会恢复内存映射并删除新文件与预览。
9. 修改正在使用的皮肤不会改写注册表，皮肤状态重新检测为未应用，并提示用户重新应用。
10. 后端 14 项测试与前端 5 项交互测试覆盖 CUR/ANI、损坏文件、同名、共享引用、交换、中文路径、回滚和键盘操作。

## 光标编辑交互修复（0.1.11，已完成）

1. 替换或分配成功后的右上角提示在 3.2 秒后自动消失，失败原因继续保留供用户查看。
2. 角色卡片和未分配文件卡片不再绑定点击事件，点击预览、名称或空白区域不会误操作。
3. 仅“替换文件”和“分配到角色”按钮触发对应操作，并保留 hover、键盘 focus 和忙碌禁用状态。
4. 前端 6 项测试覆盖按钮点击边界、键盘操作、重复提交保护和提示生命周期。

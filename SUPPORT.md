# 获取支持

Cursor Skin Manager 由维护者按可用时间维护，不提供付费客服、服务等级协议或固定响应时限。提交反馈前，请先确认问题属于本项目，并选择正确渠道。

## 选择渠道

| 问题类型 | 使用渠道 | 不要做什么 |
| --- | --- | --- |
| 应用 Bug | 搜索后创建 [GitHub Issue](../../issues) | 不要只写“打不开”而没有版本、环境和复现步骤 |
| 使用问题 | 创建标题以 `[使用问题]` 开头的 [GitHub Issue](../../issues) | 当前未开启 Discussions，不要在无关 PR 中提问 |
| 功能建议 | 创建标题以 `[功能建议]` 开头的 [GitHub Issue](../../issues) | 不要直接实现在线市场、账号、收藏、标签或云同步 |
| 安全漏洞 | 使用仓库 [Security 页面](../../security) 的 **Report a vulnerability** | 绝对不要公开漏洞细节、利用代码或敏感日志 |
| 行为准则事件 | 按照 [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) 报告 | 不要在技术讨论中公开敏感经历或个人资料 |
| 第三方光标包自身问题 | 联系皮肤包作者、发布网站或下载来源 | 不要要求本项目修复素材内容、授权或下载服务 |

## 创建 Issue 前

1. 使用最新公开版本复现问题；旧版本可能会先被要求升级。
2. 阅读 README 的常见问题，并搜索已有 Issue。
3. 记录最短复现步骤、预期结果和实际结果。
4. 准备版本、Windows 版本、系统架构，以及安装版或便携版信息。
5. 只截取 `app.log` 或 `startup.log` 中相关片段，并遮盖用户名、私人路径和其他敏感数据。

不要上传整个 `%APPDATA%\CursorSkinManager` 数据目录、真实 `library.json`、私有光标包、访问凭据或与问题无关的日志。

## 应用 Bug

以下情况通常属于本项目 Bug：

- 有效的 `.cur`、`.ani`、`.zip` 或 `install.inf` 无法导入、解析或预览。
- 应用、恢复默认、状态检测、删除或托盘行为与界面提示不一致。
- 替换角色、文件交换、备份或失败回滚导致应用内部数据错误。
- 同一问题可以在最新版本和明确的 Windows 环境中重复出现。

Issue 中请提供必要的错误信息和最小复现条件。只有在你拥有再分发权利时才能上传测试文件；否则请使用自行制作的最小样本或描述文件结构、编码和扩展名。

## 使用问题

如果功能按设计工作，但你不确定如何导入、应用、恢复默认、查看日志或退出托盘，请创建 `[使用问题]` Issue。README 已经说明的内容可能会直接链接到对应章节，重复或缺少必要信息的问题可能被关闭。

项目不提供远程控制电脑、代为整理私人皮肤库、定制光标包或恢复第三方下载服务。

## 功能建议

功能建议应说明实际问题、典型流程、受影响用户和为什么现有功能不足。当前项目范围是 Windows 当前用户的本地光标皮肤管理 MVP；在线市场、账号、收藏、标签、社交和云同步不在当前范围内。

较大功能在维护者确认前不要直接提交 Pull Request。被接受讨论不等于承诺实现或发布时间。

## 安全漏洞

不要创建公开 Issue。当前项目仅使用 GitHub Private Vulnerability Reporting，不公布个人安全邮箱。

当 [Security 页面](../../security) 显示 **Report a vulnerability** 时，请从该入口提交影响版本、攻击条件、影响范围、最小复现和缓解建议。若入口暂时不可见，请不要公开漏洞详情，等待安全策略完成配置。

普通启动失败、光标包解析错误或不包含安全影响的崩溃应作为应用 Bug 报告。

## 第三方光标包问题

应用负责本地导入、识别、预览和应用光标文件，但不维护第三方皮肤包本身。

以下问题应联系皮肤包来源方：

- 下载链接失效、压缩包密码、素材缺失或安装说明不完整。
- 光标文件在 Windows 中也无法使用、已经损坏，或 `install.inf` 本身映射错误。
- 皮肤视觉内容、角色设计、许可、署名、再分发或版权争议。
- 包中缺少某些 Windows 光标角色，但应用已经正确显示为“不完整”。

如果同一有效文件可以由 Windows 正常加载，但 Cursor Skin Manager 无法导入、预览或应用，则可能属于应用兼容性 Bug。请提供不侵犯第三方权利的最小复现信息。

## 处理原则

- 维护者会按影响、可复现性、项目范围和可用时间处理反馈，不保证回复或修复日期。
- 缺少必要信息、无法复现、超出 MVP、重复或应由第三方处理的 Issue 可能被关闭。
- 关闭不代表否定问题本身；后续有新的可复现信息时可以补充并请求重新评估。
- 修复会进入后续版本和 Release，维护者不会移动已经公开的版本标签。

希望参与修复时，请阅读 [CONTRIBUTING.md](CONTRIBUTING.md)。

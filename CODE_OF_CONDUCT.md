# 社区行为准则

Cursor Skin Manager 希望为所有参与者提供尊重、专业且可协作的社区环境。本准则适用于项目仓库、Issue、Pull Request、代码审查、Release 讨论，以及代表本项目进行的公开交流。

参与本项目即表示你同意遵守本准则。维护者同样受本准则约束。

## 我们鼓励的行为

- 尊重不同经验、背景、语言、身份和观点的参与者。
- 围绕代码、设计、文档和可复现事实讨论，不针对个人。
- 给出具体、可执行的反馈，也愿意接受合理的修改建议。
- 在犯错后承认影响、及时纠正，并帮助恢复正常协作。
- 保护日志、截图、路径和测试文件中可能出现的个人信息。
- 让讨论保持在当前 Issue 或 Pull Request 的范围内。

## 不可接受的行为

- 侮辱、贬低、歧视、骚扰、跟踪、威胁或人身攻击。
- 未经同意发布他人的身份信息、联系方式、私人路径或其他敏感数据。
- 性暗示、露骨内容、不受欢迎的性关注或针对身份特征的攻击。
- 故意扰乱讨论、重复刷屏、恶意消耗维护资源或规避已有处置措施。
- 冒充他人、伪造证据、操纵问题报告，或以报复方式对待举报者和协助调查者。
- 在项目空间传播恶意软件、凭据、无权分发的文件或明显违法内容。

合理的技术分歧、对实现的批评和维护者拒绝范围外功能，本身不构成违规；表达方式仍须保持尊重并基于事实。

## 如何报告

不要在公开讨论中披露与事件无关的隐私信息。

- 对仓库内可公开核实的违规，可创建一个标题以 `[行为准则]` 开头的 [GitHub Issue](../../issues)，只提供相关公开链接、简要事实和希望维护者采取的措施。
- 对威胁、骚扰、仇恨内容、冒充、隐私泄露等敏感或平台级问题，请使用 [GitHub Report Abuse](https://support.github.com/contact/report-abuse) 直接向 GitHub 举报。
- 如果报告涉及当前维护者，或你不适合通过公开 Issue 报告，请跳过仓库渠道并使用 GitHub Report Abuse。
- 安全漏洞不属于行为准则报告，请按照 [SECURITY.md](SECURITY.md) 私密提交，不要创建公开 Issue。

项目目前不公布个人邮箱，也没有私密的行为准则收件箱。请勿将敏感经历、住址、电话、私人账号或未公开证据写入公共 Issue。维护者不承诺固定响应时间，但会在能够处理时审查仓库内报告。

## 维护者责任

当前仓库维护者 `Myming15` 负责解释和执行本准则，包括：

- 审查可核实的上下文，并尽量让处置与行为严重程度相称。
- 删除、隐藏或编辑不适当内容，关闭失控讨论，限制或阻止持续违规账号参与。
- 在公开说明处置时尽量减少个人信息，只说明维护项目秩序所需的事实。
- 不因善意报告、提供证据或参与调查而报复任何参与者。
- 当自身存在直接利益冲突时，引导报告者使用 GitHub 平台举报渠道。

## 处置措施

维护者可根据影响、严重程度、历史行为和纠正意愿采取以下一种或多种措施，不要求每次都从最低级别开始：

1. **提醒与纠正**：指出具体问题，要求编辑或停止相关行为。
2. **正式警告**：删除或隐藏内容，明确再次发生时的后果。
3. **临时限制**：锁定讨论、暂时拒绝互动，或在一段合理时间内不接受相关参与者的贡献。
4. **永久限制**：关闭相关贡献，并通过 GitHub 的屏蔽、举报或其他仓库权限阻止继续参与。

明显威胁、人肉搜索、恶意软件、严重骚扰或持续规避处置可直接采用更高级别措施，并交由 GitHub 或有关机构处理。

## 申诉与更正

被处置者可以创建一个简短的新 Issue，请求复核事实或说明已采取的纠正措施。申诉不得重新发布被删除的敏感内容，也不得骚扰举报者。维护者可以维持、调整或撤销原处置，并在适合公开时说明结果。

本准则会随项目社区渠道的发展进行修订。重大变化将通过仓库历史公开记录。
